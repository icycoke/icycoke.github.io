MyBatis学习笔记

Posted on 2020-07-09 Edited on 2020-08-21

1. 简介

  • MyBatis是一款持久层框架
  • 支持定制化SQL、存储过程和高级映射，避免了几乎所有的JDBC代码和手动设置参数以及获取结果集
  • 采用了ORM思想，对JDBC进行了封装，屏蔽了JDBC API底层访问细节

2. ORM思想

  • ORM指Object Relational Mapping
  • ORM思想把数据库表与实体类和实体类的属性进行对应，让我们可以通过操作实体类来操作数据库表

5. 配置解析

5.1 environments（环境配置）

  • MyBatis 可以配置成适应多种环境，这种机制有助于将 SQL 映射应用于多种数据库之中

  • 尽管可以配置多个环境，但每个 SqlSessionFactory实例只能选择一种环境

  • 指定创建环境的方式

  • 通过environment元素定义环境配置

5.2 properties（属性）

  • 编写.properties文件并从配置文件中引入

5.3 typeAliases（类型别名）

  • 类型别名可为 Java 类型设置一个缩写名字，它仅用于 XML 配置，意在降低冗余的全限定类名书写

  • 配置方法

    • 直接具体配置typeAlias元素

    • 指定一个包名，MyBatis会在包名下搜索需要的JavaBean

      在该方法中，可以给实体类加上@Alias("xxx")注解自定义别名

  • Java原始数据类型为下划线+类型名，如：_int -> int

  • Java原属数据包装类型为其类型名（不区分大小写），如： int/integer -> Integer

12 缓存

12.1 缓存概述

  • 如果将用户经常查询底数据放在缓存中，则用户在查询数据时就不需要从磁盘上查询，而是从缓存中直接查询
  • 缓存可以提高查询效率，减少和数据库的交互次数，解决高并发系统的性能问题
  • 经常查询且不经常修改的数据适合加入缓存

12.2 MyBatis缓存

  • MyBatis默认定义了两级缓存：一级缓存和二级缓存
    • 默认情况下，只有一级缓存开启（SqlSession级别的缓存，也称为本地缓存）
    • 二级缓存需要手动开启和配置，是基于namespace的缓存
    • 为了提高扩展性，MyBatis定义了缓存接口Cache，用户可以通过实现Cache接口来自定义二级缓存
  • 映射语句文件中的所有 insert、update 和 delete 语句会刷新缓存

12.2.1 一级缓存

  • 一级缓存以SqlSession为单位
  • 一级缓存是默认全局开启的
  • 会话的clearCache()方法可以手动清除缓存
  • 缓存会被视为读/写缓存，这意味着获取到的对象并不是共享的，可以安全地被调用者修改，而不干扰其他调用者或线程所做的潜在修改

12.2.2 二级缓存

  • 二级缓存以命名空间为单位，一个命名空间对应一个二级缓存

  • 工作机制

    • 一个会话查询一条数据时，这条数据会被保存在当前会话的一级缓存中
    • 如果当前会话关闭了，这个会话对应的一级缓存中的数据将会被保存到二级缓存中
    • 同一命名空间下的新会话可以从该二级缓存中获取内容

  • 要启用全局的二级缓存，只需要在你的 SQL 映射文件中添加一个标签：

    可以通过修改标签的属性配置二级缓存

    这个更高级的配置创建了一个清除策略为 FIFO的二级缓存，每隔 60 秒刷新，最多可以存储结果对象或列表的 512 个引用，而且返回的对象被认为是只读的，因此对它们进行修改可能会在不同线程中的调用者产生冲突

  • 使用二级缓存时，要保证实体类实现了Serializable接口（可序列化）
