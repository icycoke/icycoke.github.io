Spring学习笔记

Posted on 2020-08-21 Edited on 2020-08-26

1. Spring简介

  • Spring是一个企业级Java应用程序开发框架
  • Spring是轻量级的，基础版本在2MB左右
  • Spring是非侵入式的，对现有的类结构没有影响
  • Spring的核心是控制反转（Inversion of Control，IoC）和面向切面编程（Aspect Oriented Programming，AOP），通过两大核心实现松耦合

3. 依赖

3.1 控制反转和依赖注入

  • 控制反转是一种思想，用IoC容器来创建对象，而不是程序主动创建依赖对象
  • 依赖注入（Dependency Injection，DI）是实现控制反转的最常见方式，由容器动态地将依赖关系注入到组件中

3.2 依赖注入方式

3.2.1 构造器注入

  • Spring默认使用无参构造器进行依赖注入

  • 可以使用多种方式通过有参构造器进行依赖注入：

    1. 参数下标

    2. 类型匹配

    3. 名称匹配

    4. 嵌套引用

3.2.2 setter注入

  • Spring同时也支持通过setter进行依赖注入

3.3 注入多种类型数据

  • 实体类

  • 通过配置文件注入

3.4 p命名空间与c命名空间

  • 实体类

3.4.1 p命名空间

  • p命名空间允许使用bean元素的属性描述Bean的属性值，以取代property元素

3.4.2 c命名空间

  • c命名空间允许使用bean元素的属性配置构造函数的参数，以取代constructor-arg元素

4. Spring配置

4.1 bean标签常用属性

4.1.1 id

  • id是bean的唯一标识符

4.1.2 class

  • class是bean对象所对应的全限定名

4.1.3 name

  • name是bean对象的别名，可以取多个别名并用空格，逗号或分号隔开

4.2 导入配置文件

  • 在beans标签内使用import标签导入其它配置文件

5. IoC容器

5.1 Bean的作用域

5.1.1 singleton - 单例作用域

  • Spring默认设置Bean的作用域为单例作用域。在此作用域下，IoC容器创建一个Bean实例，并为该Bean的后续所有请求都返回该实例的引用

5.1.2 prototype - 原型作用域

  • 设置Bean的作用域为原型作用域。在此作用域下，IoC容器为每一次对应Bean的请求都返回一个新的实例

5.1.3 其它作用域

作用域 描述
request 将单个bean作用域限定为单个HTTP请求的生命周期
session 将单个bean作用域限定为一个HTTP的Session的生命周期
application 将单个bean作用域限定为的一个ServletContext的生命周期
websocket 将单个bean作用域限定为一个Websocket的生命周期

5.2 Bean的自动装配

  • Spring可以在上下文中自动寻找并给Bean的属性赋值

  • 通过bean元素的autowire属性使用自动装配

6. 使用注解开发

6.1 前提条件

  • 必须导入约束并添加XML标签，并导入aop相关包

  • 注解注入在XML注入之前执行。因此，XML配置将覆盖通过两种方法注入的属性

6.2 常用注解

6.2.1 @Autowired

  • 可对字段、构造器、构造器参数、setter方法使用，进行自动装配

  • 默认先byType，再byName

6.2.2 @Qualifier

  • 显式地配置装配的目标，结合@Autowired使用以指定唯一的Bean对象

6.2.3 @Resource

  • 可以实现类似于@Autowired与@Qualifier的功能

  • 默认先byName，再byType

6.2.4 @Component

  • 将一个类视为组件，配合xml文件进行依赖注入

  • 衍生出@Controller、@Repository、@Service注解，四个注解的功能一致

6.2.5 @Value

  • 通过该注解给属性赋值，也可用于setter方法

6.2.6 @Scope

  • 配置Bean的作用域
