Computer Security

Posted on 2020-08-27 Edited on 2020-09-11

Intros

What is Computer Security?

  • Computer security is the protection of valued assets of a computer system
  • The value of an asset depends on the owner’s or user’s perspective
  • Also time-dependent

Basic Components of Security

Confidentiality

  • Keeping data and resources hidden
  • Only authorized people or systems can access protected data

Integrity

  • Data integrity (integrity)
  • Origin integrity (authentication)

Availability

  • Enabling access to data and services
  • Achieving availability may require concurrency control and fault tolerance

Threat Classes

Disclosure

  • Unauthorized access to information
    • Snooping: unauthorized interception of information

Deception

  • Acceptance of false data
    • Modification, spoofing, repudiation of origin, denial of receipt

Disruption

  • Interruption or prevention of correct operations

Usurpation

  • Unauthorized control of part of a system
    • Modification, spoofing, delay, denial of service

General Categories of Security Attack

  • Spoofing attack (Disclosure)

    • IP address spoofing
    • ARP spoofing
    • DNS spoofing

  • Snooping/Sniffing (Deception, Disruption, Usurpation)

  • Modification/Alternation (Deception, Usurpation)

  • Repudiation of origin (Deception)

  • Denial of receipt (Usurpation)

  • Delay (Usurpation)

  • Denial of service (Usurpation)

Goals of Security

  • Prevention
    • Prevent attackers from violating security policy
  • Detection
    • Detect attackers’ violation of security policy
  • Recovery
    • Stop attack, assess and repair damage
    • Continue to function correctly even if attack succeeds

Policy and Mechanism

Policy

  • Policy says what is, and is not ,allowed
  • Policy defines “security” for the system/site/etc.
  • If policies conflict, discrepancies may create security vulnerabilities

Mechanism

  • Mechanisms enforces policies
  • Types of mechanisms
    • Secure
    • Precise
    • Broad
  • Trusts on the mechanisms
    • Each mechanism implements part of the security policy
    • The union of security mechanisms implements all aspects of the security policy
    • The mechanisms are implemented correctly
    • The mechanisms are installed and administered correctly

Assurance

  • Provide a basis of “how much” to trust a system
  • Specification
    • Requirements analysis
    • Statement of desired functionality
  • Design
    • How system will meet specification
  • Implementation
    • Programs/systems that carry out design

Operation Issues

  • Cost-benefit analysis
    • Is it cheaper to prevent or recover?
  • Risk analysis
    • Should we protect something?
    • How much should we protect this thing?
  • Laws and customs
    • Are desired security measures illegal?
    • Will people do them?

Access Control

  • An access control mechanism is used to permit a subject to access an object in a particular mode
  • Unauthorized subjects are not permitted to access the object

Terminology

  • Subject: human user or program running on their behalf
  • Object: thing on which an action can be performed
  • Access modes: any controllable actions of subjects on objects
